  <action>
Modify src/components/stops/StopPane.svelte to add Svelte fade transitions on arrival list items.

1. Add import at the top of the script block (after the existing imports):
   ```javascript
   import { fade } from 'svelte/transition';
   ```

2. In the template, wrap each AccordionItem inside the `{#each}` block with a `<div>` that carries the fade transitions. Replace the current block (around lines 250-263):

   FROM:
   ```svelte
   {#each arrivalsAndDepartures.arrivalsAndDepartures as arrival (makeKey(arrival))}
     <AccordionItem data={arrival} data-new={arrival._isNew}>
   ```

   TO:
   ```svelte
   {#each arrivalsAndDepartures.arrivalsAndDepartures as arrival (makeKey(arrival))}
     <div in:fade={{ duration: 300 }} out:fade={{ duration: 200 }}>
       <AccordionItem data={arrival} data-new={arrival._isNew}>
   ```

   And close the wrapper `</div>` after the `</AccordionItem>` closing tag.

   The resulting structure should be:
   ```svelte
   {#each arrivalsAndDepartures.arrivalsAndDepartures as arrival (makeKey(arrival))}
     <div in:fade={{ duration: 300 }} out:fade={{ duration: 200 }}>
       <AccordionItem data={arrival} data-new={arrival._isNew}>
         {#snippet header()}
           <span>
             <ArrivalDeparture arrivalDeparture={arrival} />
           </span>
         {/snippet}
         <TripDetailsPane
           {stop}
           tripId={arrival.tripId}
           serviceDate={arrival.serviceDate}
         />
       </AccordionItem>
     </div>
   {/each}
   ```

WHY this approach works:
- Svelte's keyed `{#each}` block triggers `in:` transitions when a new key enters and `out:` transitions when a key leaves
- Items with the same key that persist between renders get NO animation — exactly matching TRAN requirement that unchanged buses do not animate
- Using separate `in:fade` (300ms) and `out:fade` (200ms) gives different durations per CONTEXT.md decisions
- The wrapper div is needed because transitions must be on a direct DOM element, not a component tag (Svelte limitation)
- The `{#key arrivalsAndDepartures.stopId}` block that wraps the Accordion means navigating to a new stop re-mounts everything — all arrivals at the new stop will fade in, which is correct UX

After making the change, run:
- `npm run build` to verify no build errors
- `npm run test` to verify no test regressions

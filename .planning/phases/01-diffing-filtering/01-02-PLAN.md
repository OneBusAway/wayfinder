**3. Modify loadData function (lines 44-73)**
After line 63 (`arrivalsAndDepartures = data.data.entry;`), replace the direct assignment with diffing logic. The modified section should be:

Replace:
```javascript
arrivalsAndDepartures = data.data.entry;
```

With:
```javascript
const entry = data.data.entry;
const rawArrivals = entry.arrivalsAndDepartures || [];
const diffed = diffArrivals(previousArrivals, rawArrivals, Date.now());
previousArrivals = rawArrivals;
arrivalsAndDepartures = { ...entry, arrivalsAndDepartures: diffed };
```

This does three things:
- Runs the diff (filters departed, tags _isNew) on the raw API response
- Stores the raw (unfiltered) arrivals as `previousArrivals` for the next poll comparison
- Rebuilds the entry object with the diffed arrivals so the rest of the component works unchanged

**4. Reset previousArrivals on stop change**
In the `$effect` block (lines 84-89) that calls `resetDataFetchInterval`, add a reset:

Change:
```javascript
$effect(() => {

});
```

To:
```javascript
$effect(() => {
  if (stop?.id) {
    clearInterval(interval);
    previousArrivals = null;
    resetDataFetchInterval(stop.id);
  }
});
```

This ensures that when navigating to a different stop, all arrivals at the new stop are tagged `_isNew = true` (first poll behavior).
